python_参数传递方式

python中一切皆为对象

对象

数字是对象，列表是对象，函数也是对象，任何东西都是对象

变量

变量是对象的一个引用（又称为名字或者标签），对象的操作都是通过引用来完成的

变量本身没有类型信息，类型信息存储在对象中

赋值操作 = 就是把一个名字绑定到一个对象上。就像给对象添加一个标签

函数参数

本质上应该是传递的对象，根据对象的类型来确定传递的方式：

  • 如果是不可变的对象则可以理解为按值传递
  • 是可变对象则可以理解为按引用传递

python基本数据结构和底层实现原理

List的实现原理

List 是一种元素个数和数据可变的线性表，可以加入和删除，在各种操作中维持已有元素的顺序．

线性表：具有元素顺序关系的某类数据对象的集合，支持创建，索引，动态改变表，两个表的操作

线性表的实现：

  1. 顺序表：将表中元素顺序底存放在一大块连续的存储区域，元素间的关系由它们的存储顺序自然表示
  2. 链接表：将表元素存放在通过链接构造起来的一系列的存储块里，一般存在单链表，循环单链表，双链表

顺序表结构：

  1. 一体式结构:将表的全局对象（表的容量max和当前已存储的元素个数num）和数据存储区以连续的方式放置在一块连续的存储区

  2. 分离式结构：　表对象只保存容量和元素个数，以及指向存储区域的链接引用对象，使得两者的存储相分离，通过链接来连接表对象和存储数据对象（创建管理较复杂）

基于分离式线性表的存储区扩容：

  • 申请一块更大的元素存储区域
  • 把原先表中的元素复制到新的存储区域
  • 改变表对象的元素区链接
  • 实际加入新的元素

python的list的实际策略：

List 采用分离式技术实现的动态顺序表

　　在建立空表或者很小的表的时候，系统分配一块能够容纳８个元素的存储区域，在执行插入操作时，如果元素区满就换一块４倍大的存储区域，但是如果表很大时，换存储去时容量加倍
，后面减少倍数，是为了避免出现过多空闲的存储位置．

list操作的基本性质

len()的时间复杂度为Ｏ（１）

元素按索引访问和赋值，尾端加入和删除时间复杂度为Ｏ（１），根据相应的索引，能够直接计算出相应的内存地址

一般位置元素的加入，切片，拼接的时间复杂度Ｏ（n）

特殊：　clear(),reverse(),sort(排序算法相关)

Turple的实现原理

元素数据不可变的线性表，一旦创建就不能更改，具有顺序结构

python的turple的实现策略：

　　Turple采用一体式结构的线性表，按建立时确定的元素个数来分配内存，一旦创建就不能进行扩容

String的实现原理

字符串是字符的线性序列，采用线性表的各种实现技术实现

python的str的实现策略：

字符串是不可变的类型，采用一体式顺序表实现

python Redis的pub/sub相关

redis简介

它是一个速度很快的内存中的非关系的数据库(NoSQL),可以存储键与５种数据结构之间的映射，分别为string,list,set,hash,zset.
它可以作为缓存和消息中间件
它支持事务，订阅发布，LUA脚本，过期自动删除key, 内存回收，自动故障转移

linux下的redis的安装

　　redis是一个服务端和客户端的模型，需要分别进行安装

  1. redis-server服务端的安装，默认情况下安装后会成为开机自启动的service，默认的主机名为localhost,端口号为6379

  2. python支持的redis-py客户端安装

  3. Redis 由四个可执行文件：redis-benchmark、redis-cli、redis-server、redis-stat 这四个文件，加上一个redis.conf就构成了整个redis的最终可用包．

　　可以通过redis-cli来进行相应的调试，用于模拟客户端的操作，在实际使用的过程中，服务端有个默认的同时连接客户端最大数量限制．所以在不用时，需要断开连接．

Redis的发布订阅Pub/sub相关

  • Redis 发布订阅(pub/sub)是一种消息通信模式：发送者(pub)发送消息，订阅者(sub)接收消息。Redis客户端可以订阅任意数量的频道，这样极大的解耦了消息的传输方式．

  • 发布消息端只需要将相应的消息publish至相应的管道（具有唯一名来进行标识不同的管道）

  • 订阅消息端只需要订阅相应的管道即可，只要一旦有消息存在，即可进行获取，通常用于多路数据分发和广播的场景中．

python_编码规范

禁用分号

  1. 每行的表达式结尾禁止使用分号

缩进Indentation

  1. 代码以４个空格来进行缩进，禁止混合使用空格和tab,除非进行了相应的配置
  2. 每个语句占一行

每行不超过80个字符

  1. 除去长的模块导入语句和注释里的URL,Python会将圆括号, 中括号和花括号中的行隐式的连接起来
  2. 注意与使用反斜线的区分，主要用于不能进行自行连接的多行操作，例如with和assert

二进制运算符可以位于段行前或者后

空行

  1. 顶级定义，即顶级函数和类定义之间空两行
  2. 类内部的函数定义空一行
  3. 函数内部使用空行来进行相应的逻辑分割

空格

  1. 括号内没有空格
  2. 在逗号，冒号和分号前无空格，必须加在符号后面
  3. 参数列表, 索引或切片的左括号前不应加空格.
  4. 在二元操作符两边都加上一个空格
  5. 当’=’用于指示关键字参数或默认参数值时, 不要在其两侧使用空格.
  6. 不要用空格来垂直对齐多行间的标记

｀#!｀的作用

  1. 类Unix操作系统的程序载入器会分析＃！后的内容, 将这些内容作为解释器指令, 并调用该指令, 并将载有Shebang的文件路径作为该解释器的参数.

导入模块import

  1. 不同模块分行进行导入
  2. 导入需要放置于文件说明前，导入次序为：标准模块，第三方模块，自定义模块
  3. 推荐使用绝对导入,支持相对路径导入（python3已经废弃）
  4. 尽量避免通配符＊导入

字符串支持单引号和双引号

使用英文及时跟新注释comments，主要是块注释和行内注释，以及文档注释docstring

命名规则name

  1. 变量名应该反映的是使用意义usage，而不是其实现过程
  2. 类名采样CapWords的命名规则，使用名词，表示抽象的集合
  3. 用单下划线(_)开头表示模块变量或函数是protected的(使用import * from 时不会导入相应的变量和函数)
  4. 用双下划线(__)开头的实例变量或方法表示类内私有
  5. 双下划线开头并结尾的名称具有python内置的特殊含义，例如__init__,__name__,__all__
  6. 函数命名一般采用动词表示完成的功能简写，词间用下划线分割,且一个函数完成一个功能
  7. 避免使用单字符进行命名操作

main函数

一个模块中的主要执行部分，通常用于进行相应的单元测试

模块和包和库的区别

  1. 模块module：　即一个.py文件，包含相应的组织代码
  2. 包package：　即一个有层次的文件目录，一定存在一个inin文件，以及其他的模块和子包
  3. 库lib：　python库是参考其它编程语言的说法，就是指python中的完成一定功能的代码集合，供用户使用的代码组合。在python中是包和模块的形式

字符串格式化 %与format

类继承

  1. 如果一个类不继承自其它类, 就显式的从object继承(新式类). 嵌套类也一样
  2. 类的多重继承：如果多个类具有相同的方法，则会以第一个类的方法为子类所使用的方法，djando rest内的各个类的继承关系
  3. 类是一切事物的抽象，包含＿＿init__用于进行实例的初始化操作，可以存在一些＠property和＠staticmethod等特殊的语法糖
  4. 双下划线开头的变量或方法为私有变量，只能在类的内部使用
  5. 类方法定义必须包含的第一个参数是self
  6. 类的专用方法
